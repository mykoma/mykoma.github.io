从上面的代码可以看出, Objective-C 的对象, 实际上对应的就是 C 里面的 struct 结构体.

而 User 类里面的 aStr 和 bStr 的属性, 也对应了 C 里面的 static 函数, 其中
这里比较重要的是 set 函数_I_User_setAStr_和_I_User_setBStr_

注意这里的_I_User_setAStr_里面调用了objc_setProperty去设置值,
而_I_User_setBStr_是直接计算偏移量去设置.

所以从效率上来说,_I_User_setBStr_的效率比_I_User_setAStr_更高,
毕竟直接使用偏移量去设置内存速度更快, 多调用一层函数, 就会多消耗一点资源, 毕竟积少成多.

结论

综上所述, 对于 NSString 的 property 来说, 使用 strong 其实会更好的一些, 底层的调用效率会更高; 不过对于现在的 cpu 来说, 两者差别不大, 如果你是一个追求极致的人, 使用 strong 或者更适合你.
